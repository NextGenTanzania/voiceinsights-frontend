## Exact changes made
1. Added public read-only Phase 20 endpoint:
   - `GET /api/public/demo-reports/:id/experience`
   - Hard-filtered with `is_demo = 1 AND status = 'published'`.
   - Returns only the safe public Phase 20 experience package.
   - Performs no mutation and does not expose organization-owned reports.

2. Kept `scripts/enrich-demo-reports.js` consistent with deployed routes:
   - Authenticated mutation remains on `/api/reports/:id/trust/enrich`.
   - Public verification reads `/api/public/demo-reports/:id/experience`, which now exists.

3. Updated `site/sample-report-viewer.html` to visibly render Phase 20:
   - One-page executive brief.
   - Procurement-grade infographic pages.
   - Methodology transparency panel.
   - Evidence panel.
   - Quality gate status.
   - Report assistant actions.
   - Executive decisions required.

4. Tightened evidence classification:
   - Quote-only demo evidence is no longer labelled as raw-source evidence.
   - Raw-source evidence now requires an auditable pointer such as response ID, transcript, audio, consent, or explicit raw-response evidence.
   - Quote-only demo evidence is labelled synthetic demo evidence.

5. Improved public export safety:
   - Public viewer downloads are now polished text exports rather than raw JSON-style object dumps.
   - Statistical annex and dataset appendix buttons now download `.txt` in the public viewer.
   - Human-readable export formatting replaces raw `JSON.stringify` object output in text reports.

6. Public wording safety:
   - Export text replaces “Not enough data” with “Insufficient verified evidence available.”
   - Phase 20 fallback messaging avoids system/debug wording.
