# VoiceInsights Africa — v182.1 Export Quality & Public Polish Patch

## Scope
Patch-only update for v182 VRDS Showcase Experience. No homepage, brand identity, navigation, authentication, database schema, API contract, or unrelated dashboard redesign was performed.

## Exact changes made

### 1. Export quality and wording
- Public viewer export labels were changed from over-promising PDF/PPTX wording to honest client-facing labels:
  - Print-ready report
  - Presentation-ready board outline
  - Export Previews
  - Executive intelligence brief
  - Donor impact brief
  - Policy brief preview
  - Infographic report preview
  - Statistical annex preview
- Public export downloads now use a sanitized, readable export-preview text structure instead of opening raw JSON-like API output in the sample library.
- Export text sanitizes unsafe phrases such as “Not enough data,” “generated by system,” “raw JSON,” “undefined,” “null,” and “NaN.”

### 2. Public wording cleanup
- Replaced internal language in the viewer:
  - “VRDS Showcase Experience” → “Executive Intelligence View”
  - “Phase 20 Procurement-Grade Experience” → “Procurement-Ready Report Experience”
  - “Structured assistant actions…” → “Report assistant actions…”
- Replaced weak fallback with: “This report experience is temporarily unavailable. Please return to the sample library or request a demo.”

### 3. Public track-download hardening
- The public download counter update now requires both `is_demo = 1` and `status = 'published'`.

### 4. Infographic polish
- Added publication-style infographic layout hooks: stronger two-column page structure, headline/interpretation/decision fields, evidence label area, print-safe spacing, and less card clutter.

### 5. Manual QA support
- Added a manual QA checklist for browser, mobile, print/PDF, export, public wording, and route safety review.
